Signal Sieve
← App openapi.json GitHub
Receipt clerk, not oracle. Signal Sieve is a heuristic pre-belief filter — it audits signal quality (source custody, pressure language, certainty inflation, evidence shape), not factual accuracy. overall_confidence counts signal markers, not truth probability. Scroll down for the interactive spec, or copy a curl below to start immediately.

Quick start

Analyze text

POST /api/v1/analyze
curl -s -X POST https://signal-sieve.onrender.com/api/v1/analyze \
  -H "Content-Type: application/json" \
  -d '{"text":"According to the BLS, CPI rose 2.4% in October 2024. The Fed held rates at 5.25–5.50%.", "source_type":"auto"}'

Capsule — minimal token response (~150 tokens, ideal for AI pipelines)

POST /api/v1/analyze.capsule
curl -s -X POST https://signal-sieve.onrender.com/api/v1/analyze.capsule \
  -H "Content-Type: application/json" \
  -d '{"text":"...", "source_type":"auto"}'

Cross-document comparison — detect numeric conflicts across sources

POST /api/v1/compare
curl -s -X POST https://signal-sieve.onrender.com/api/v1/compare \
  -H "Content-Type: application/json" \
  -d '{
  "documents": [
    {"text": "10-Year Treasury Yield: 4.61%  WTI Crude: $68.43", "source_name": "Schwab"},
    {"text": "10-year Treasury yield: 4.56%.  Brent crude: $72.10.", "source_name": "BlackRock"}
  ]
}'

Plain-text report (stable section headers)

POST /api/v1/analyze.txt
curl -s -X POST https://signal-sieve.onrender.com/api/v1/analyze.txt \
  -H "Content-Type: application/json" \
